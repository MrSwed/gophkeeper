
Клиент
 - шифрование:
   - ключ для шифрования генерируется случайным образом,
     шифруется парольной фразой (текущими username и паролем * ),
     при смене пароля или логина, перезашифровывается  с новым паролем и
     синхронизируется на сервер в зашифрованном виде, храниться в
     конфигурационном файле пользователя (packed_key)
   - gzip, потом шифрование
   - расшифровка, ungzip
 - хранит зашифрованные данные локально
 - синхронизация с сервером
   - по запросу
   - авторизация, логин/пароль, потом клиентский токен
 - настройки
   - расположение настроек по умолчанию в системе "~/.config/GophKeeper"
   - автоматические
     - путь к базе пользователя sqlite
     - путь к локальному хранилищу (по умолчанию в системе "~/.config/GophKeeper/<profile>/"
     - данные синхронизации
     - дата обновления конфигурационного файла
   - настраиваемые через аргумнты командной строки
     - имя профиля
     - email
     - адрес (с портом) сервера grpc
 - шаблоны типов данных
   - пара логин/пароль
   - текст
   - файл
   - банк.карта
 - мета поля данных
   - дата создания
   - дата последнего изменения
   - уникальный ключ-идентификатор
   - описание (не шифруемый текст)
 - синхронизация с удаленным сервером
   - хранение app_token - ключ приложения, зашифрован основным ключем шифрования

 - одновременная работа нескольких клиентов локально (?)
 - режим работы демона (?)

Сервер
  - регистрация нового приложения:
    email, пароль аутентификации на сервере,
    отличается от парольной фразы, также запрашивается у пользователя с консоли
    - известный email:
      - ОК, пароль введен верно
      - Ошибка, пароль неверный
    - новый email:
      - ОК, регистрация, если email с паролем проходят валидацию
      - Ошибка, регистрация, если email с паролем не проходят валидацию
    - ответ:
      - app token, токен для подключаемого приложения (шифруется ключом
        шифрования пользователя и сохраняется в файле конфигурации)
  - синхронизация
    - token приложения в metadata
    - key - уникальный ключ для работы с записями
    - новые по update_at
    - признак удаления - отсутствие blob (зашифрованного пакета)
